Antigravity 簡介：Google 雲端 IDE

Antigravity 是什麼？
Google 推出的完全雲端 AI 程式設計環境（Project IDX 升級版）。無需本地配置，直接在瀏覽器中開發、測試、部署。內建 AI 助手，支援多種框架。

Antigravity 的獨特優勢

完全雲端

無需配置環境，打開瀏覽器即用

AI Agent Mode

AI 自動執行任務，無需手動干預

自動測試

邊寫邊測，問題立即反饋

一鍵部署

直接部署到 Netlify、Firebase

團隊協作

分享連結即可多人同步編輯

Google 生態

與 Google Cloud、Firebase 無縫整合

Antigravity vs Cursor/Windsurf

Cursor / Windsurf

• 本地 IDE
• 需要配置環境
• 完全離線工作
• 代碼在你的電腦

Antigravity

• 雲端 IDE
• 零配置，即開即用
• 需要網絡連接
• 代碼在 Google Cloud

適用場景

  • 快速原型：5 分鐘創建完整應用
  • 無設備限制：平板、Chromebook、舊電腦都能用
  • 團隊協作：即時共享代碼，一起編程
  • 自動化任務：用 AI Agent 自動完成重複工作
  • 學習環境：初學者無需複雜配置
注意事項
Antigravity 需要 Google 帳號和網絡連接。如果你注重隱私或經常離線，本地 IDE（Cursor/Windsurf）更合適。

Antigravity 開始使用與快捷鍵

開始使用

  1. 訪問官網：https://antigravity.google
  2. 使用 Google 帳號登入：直接用現有 Google 帳號即可
  3. 建立新專案：選擇「New Project」開始新專案
  4. 選擇專案模板：可選擇 React、Vue、Angular、Flutter 等框架模板
  5. 開始編碼：進入雲端編輯環境，直接開始開發
零配置的優勢
Antigravity 是完全雲端的 IDE，無需下載安裝任何軟體。只要有瀏覽器和網路連線，任何裝置都能使用，包括 Chromebook、平板電腦、甚至舊電腦。

桌面應用（可選）

若需要更接近本地 IDE 的體驗，可下載桌面版：

  • 下載網址：https://antigravity.google/download
  • 支援系統：Windows、macOS、Linux
  • 優勢：更快的啟動速度、離線編輯（需同步）、系統整合更佳

常用快捷鍵

Cmd/Ctrl + ` → 開啟/關閉終端機 Cmd/Ctrl + L → 開啟 AI Agent 面板 Cmd/Ctrl + E → 切換 Editor/Manager 視圖 Cmd/Ctrl + I → 開啟指令面板 Cmd/Ctrl + P → 快速開啟檔案 Cmd/Ctrl + S → 儲存並自動同步

AI 功能快捷鍵

【程式碼生成】 Cmd/Ctrl + Shift + G → 生成測試 Cmd/Ctrl + Shift + D → 生成文檔 Cmd/Ctrl + Shift + R → 重構選中代碼 【協作功能】 Cmd/Ctrl + Shift + S → 分享專案 Cmd/Ctrl + Shift + C → 開啟協作模式 Cmd/Ctrl + Shift + P → 發佈/部署

介面說明

區域 功能說明
左側面板 檔案瀏覽器、搜尋、版本控制
中央區域 程式碼編輯器（支援多分頁）

小技巧
第一次使用建議先從範本專案開始，熟悉介面後再建立空白專案。Antigravity 的 AI Agent 面板（Cmd/Ctrl + L）是最強大的功能，可以用自然語言描述需求，AI 會自動完成任務。

Antigravity Agent Mode：AI 自動開發

Agent Mode 的革命性意義

從"我寫代碼"到"AI 寫代碼"
傳統開發：開發者逐行編寫
協作編程：開發者與 AI 對話，AI 幫忙補全
Agent Mode：你設定目標，AI 自動完成所有工作，包括編寫、測試、修復 Bug

Agent Mode 的工作流

  1. 設定目標："建立一個天氣查詢應用"
  2. 指定需求：使用 React、API 源、設計要求等
  3. 啟動 Agent：點擊"Run Agent"按鈕
  4. AI 自動執行：
    • 創建項目結構
    • 安裝依賴
    • 生成所有代碼
    • 運行測試
    • 修復問題
  5. 審查結果：檢查 AI 的成果，需要改進時修正

Agent Mode 實例

目標："用 Claude API 做一個聊天機器人"

Agent 的自動任務清單：
✓ 初始化 Node.js 項目
✓ 安裝 claude-sdk
✓ 生成主程序代碼
✓ 配置 API Key
✓ 創建測試用例
✓ 運行測試
✓ 發現問題 → 自動修復
✓ 完成！（5 分鐘內）

傳統方式需要 1-2 小時

Agent Mode vs Manual Mode

特性 Agent Mode Manual Mode
開發速度 最快（自動化一切） 較慢（需要手動指導）
控制力 低（AI 主導） 高（開發者主導）
學習效果 較差（看不到過程） 較好（逐步理解）
適用場景 快速原型、重複任務 學習、精細化開發

何時使用 Agent Mode？

  • 你很明確知道要什麼，不需要邊走邊改
  • 這是一個重複性的、標準化的項目
  • 時間緊張，需要快速完成
  • 你已經熟悉開發流程，只是想加快速度

何時不用 Agent Mode？

  • 你在學習編程，需要理解每個步驟
  • 這是複雜的商業邏輯，需要精細控制
  • 需要多次迭代和調整

Antigravity 瀏覽器整合與自動測試

內建瀏覽器預覽

邊寫邊看效果
Antigravity 的左側是代碼編輯器，右側是即時預覽瀏覽器。修改代碼時自動刷新，看到變化。無需在多個窗口之間切換。

自動測試系統

  1. 智能測試建議：Antigravity 會根據你的代碼自動建議測試用例
  2. 一鍵運行：點擊"Run Tests"，自動執行所有測試
  3. 實時反饋：測試結果即時顯示，包括覆蓋率、錯誤位置
  4. 自動修復：如果測試失敗，AI 會建議修復方案

測試工作流示例

1. 你寫了一個計算函數： function add(a, b) { return a + b; } 2. Antigravity 自動建議測試： • add(1, 2) 應返回 3 • add(-1, 1) 應返回 0 • add(0, 0) 應返回 0 3. 你批准測試 → 自動運行 4. 所有測試通過 ✓ 5. Antigravity 提示："代碼覆蓋率 100%"

除錯功能

  • 即時堆棧追蹤：錯誤發生時立即顯示調用堆棧
  • 變數檢查：暫停執行時查看所有變數值
  • 條件斷點：設定斷點的觸發條件
  • AI 自動除錯：AI 分析錯誤並建議修復

實時協作測試

團隊開發場景
• 分享 Antigravity 連結
• 多人同時編輯和測試
• 所有測試結果實時同步
• 一人修改代碼，所有人即時看到效果

性能分析

  • 內建性能分析工具
  • 實時監控 CPU、記憶體、執行時間
  • AI 建議優化方案
提示
Antigravity 的自動測試功能非常強大，但可能無法覆蓋所有邊界情況。重要的代碼還需要手動測試驗證。

Antigravity 部署到 Netlify 實戰

為什麼選擇 Netlify？

  • 免費，支援自動部署
  • 無需設定伺服器，自動 HTTPS
  • 與 GitHub 深度整合
  • 自動回滾和版本管理
  • 全球 CDN，訪問速度快

部署步驟（非常簡單）

Step 1：在 Antigravity 中完成開發

確保代碼能正常運行，所有測試通過。

Step 2：推送到 GitHub

1. 在 Antigravity 右上角點擊 "Push to GitHub" 2. 授權 Antigravity 訪問你的 GitHub 3. 選擇倉庫或創建新倉庫 4. 點擊 Confirm

Step 3：在 Netlify 連接

  1. 訪問 https://netlify.com
  2. 用 GitHub 帳號登入
  3. 點擊 "New site from Git"
  4. 選擇你剛推送的倉庫
  5. 設定構建命令和輸出文件夾

Step 4：自動部署

完成！
Netlify 會自動：
✓ 監聽 GitHub 更新
✓ 每次 Push 時自動部署
✓ 生成唯一 URL（https://xxx.netlify.app）
✓ 配置 HTTPS 證書
✓ 設定自動備份

部署配置範例

（Netlify 自動檢測） React 項目： - Build command: npm run build - Publish directory: build Next.js 項目： - Build command: npm run build - Publish directory: .next Vue 項目： - Build command: npm run build - Publish directory: dist

部署後的管理

功能 說明
自定義域名 綁定自己的域名
環境變數 安全設定 API Key、數據庫連接
版本管理 回滾到之前的部署版本
Webhook 與第三方服務整合
分析 查看訪問量、性能指標

實戰建議

  • 第一次部署可能需要 5-10 分鐘，之後每次更新只需 1-2 分鐘
  • 如果部署失敗，檢查 Netlify Build 日誌找出問題
  • 環境變數要在 Netlify Dashboard 中設定，不要硬寫在代碼裡
  • 定期檢查依賴更新，保持安全性
