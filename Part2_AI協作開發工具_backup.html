什麼是 AI 協作編程？

AI 協作編程定義
不是單純的代碼自動生成，而是人與 AI 的深度協作。開發者用自然語言描述需求，AI 理解上下文並主動提示、建議、生成代碼，開發者驗證、修改並指導 AI 改進。

Cursor IDE 簡介與特色

基於 VSCode 構建的最強 AI 協作編程 IDE。完全相容 VSCode 擴展，同時深度整合多個 AI 模型（GPT-4、Claude、Gemini），提供無與倫比的代碼智能。

Cmd/Ctrl + L

邊寫邊有 AI 幫你完成，像心靈感應一樣

引用當前工作區的任何函數、類、變數

Codebase AI

AI 理解整個專案結構，實現真正的全局優化

Chat

邊修改代碼邊對話，實時反饋與修改

多模型支援

GPT-4、Claude 3.5、Gemini 隨時切換

  • 學習最快：無縫的 AI 體驗，幾分鐘就能上手

  • 社群活躍：大量教學、模板、最佳實踐共享
  • 功能持續更新：每週新功能，快速迭代

核心功能：AI 代碼生成

Cmd/Ctrl + K 實踐

使用場景
想要生成某段代碼時，選中代碼區域或放置光標，按 Cmd/Ctrl + K，用自然語言描述需求，Cursor 會生成完整代碼。

實例 1：生成函數

輸入提示： "寫一個函數，接收數字陣列，返回所有偶數的平方和"

function sumSquareEvenNumbers(arr) { return arr .filter(num => num % 2 === 0) .reduce((sum, num) => sum + num * num, 0); }

實例 2：生成 React 元件

輸入提示： "寫一個 React TodoList 元件，有添加、刪除、完成任務的功能"

import { useState } from 'react'; export default function TodoList() { const [todos, setTodos] = useState([]); const [input, setInput] = useState(''); const addTodo = () => { if (input.trim()) { setTodos([...todos, { id: Date.now(), text: input, done: false }]); setInput(''); } }; return ( <div className="p-6"> <h1>我的待辦清單</h1> {/* 顯示和管理任務的代碼 */} </div> ); }

進階技巧：使用 @symbols

@codebase 的魔力
在提示中輸入 @ 可快速引用已有的函數、類、變數。例如："使用 @calculateTotal 函數計算總價" 會自動引用該函數的完整定義，確保 AI 生成的代碼相容。

生成質量提升秘訣

  • 提示越詳細越好：包括輸入、輸出、邊界條件
  • 提供例子：「例如 [1,2,3] 應返回 14」會大幅提升準確度
  • 指定語言/框架：「用 React Hooks 和 Tailwind CSS」
  • 描述設計模式：「使用 MVC 架構」或「遵循 SOLID 原則」

核心功能：AI 代碼修改與重構

快速修改流程

Cmd/Ctrl + K 也用於修改
選中現有代碼，按 Cmd/Ctrl + K，輸入修改需求（如"優化性能"、"加上錯誤處理"、"轉換為異步"），Cursor 直接在原位置修改。

常見修改需求

優化性能

"優化此函數的時間複雜度"

錯誤處理

"加入 try-catch 與驗證"

單元測試

"為此函數寫 Jest 測試"

代碼重構

"轉換為函數式編程風格"

文檔化

"加上 JSDoc 註釋"

語言轉換

"將此代碼轉換為 Python"

實例：代碼優化

原始代碼：

function findDuplicates(arr) { const duplicates = []; for (let i = 0; i < arr.length; i++) { for (let j = i + 1; j < arr.length; j++) { if (arr[i] === arr[j]) { duplicates.push(arr[i]); } } } return duplicates; }

優化後（選中代碼 → Cmd/Ctrl+K → "優化性能"）：

function findDuplicates(arr) { const seen = new Set(); return [...new Set(arr.filter(item => seen.has(item) || !(seen.add(item) || false) ))]; }

批量重構提示

  • "將此檔案中的所有 var 改為 const"
  • "為所有 API 呼叫加上超時設定"
  • "將回調函數轉換為 Promise/async-await"
  • "提取魔法數字為具名常數"

GitHub Copilot 簡介與六大功能

GitHub Copilot 是什麼？
由 GitHub 和 OpenAI 合作開發，基於 Codex 模型的代碼自動補全工具。深度整合 GitHub 工作流，支援所有主流 IDE（VS Code、JetBrains、Visual Studio、Neovim）。

六大核心功能

1. 自動補全

即時預測並補全代碼，從單行到整個函數，大幅提升編寫速度

2. Copilot Chat

在 IDE 中與 AI 對話，詢問問題、解釋代碼、討論架構設計

3. 註解驅動開發

撰寫註解描述需求，Copilot 自動生成對應的完整代碼實現

4. 單元測試生成

自動為函數生成測試用例，提升代碼品質與覆蓋率

5. 程式碼重構

智能建議代碼優化方案，改善效能與可讀性

6. 文件生成

自動生成 JSDoc、docstring 等文檔註釋，保持代碼文檔同步

支援環境

IDE 支援程度 特色

Visual Studio ✓ 完整支援 適合 .NET 開發者
JetBrains 系列 ✓ 完整支援 IntelliJ、PyCharm、WebStorm 等
Neovim ✓ 支援 終端機愛好者首選

實例 2：註釋生成代碼

function calculateAge(birthDate) { const today = new Date(); const age = today.getFullYear() - birthDate.getFullYear(); const monthDiff = today.getMonth() - birthDate.getMonth(); if (monthDiff < 0 || (monthDiff === 0 && today.getDate() < birthDate.getDate())) { return age - 1; } return age; }

  • 循序漸進：一步步編寫，每次一個小功能

Copilot X 的新功能
在 IDE 中開啟對話面板，像 ChatGPT 一樣與 Copilot 對話，但有代碼上下文。可以問問題、解釋代碼、生成測試、討論架構。

Windsurf Flow Mode：自然對話編程

什麼是 Flow Mode？

Flow Mode 的核心理念
打破人與 AI 的對話邊界。不是一句一句指令，而是自然對話式的合作。你說一個大方向，AI 會自動處理細節，邊處理邊向你報告進度。

Flow Mode vs 傳統 Chat 的區別

方面 Flow Mode 傳統 Chat（Cursor/Copilot）
交互方式 持續對話，實時修改 發送指令，等待回應
代碼修改 AI 主動修改，自動同步 生成代碼片段，手動複製
上下文理解 自動理解整個流程 需要明確指定相關文件
錯誤恢復 AI 自動檢測和修復 需要手動指出錯誤

Flow Mode 實例對話

場景：建立用戶認證系統

你："我要加一個登入系統，用 JWT。"
Windsurf："好，我會：
1. 在後端加 JWT 驗證邏輯
2. 修改數據庫模型加密碼欄位
3. 前端加登入頁面
4. 加上錯誤處理"
（Windsurf 自動同時修改所有文件）

你："再加一個記住我功能"
Windsurf："已添加，更新了 token 刷新邏輯"

（無需你指定哪些文件要改，AI 全部自動協調）

啟用 Flow Mode

  1. 在 Windsurf 中打開 Chat 面板
  2. 點擊 "Flow Mode" 按鈕（左上角）
  3. 開始自然對話

Flow Mode 的最佳實踐

  • 描述"做什麼"而不是"怎麼做"
  • 讓 AI 自主決定實現方式
  • 邊走邊提出新需求，AI 會自動適應
  • 信任 AI 的決定，除非明確需要改進

Antigravity 部署到 Netlify 實戰

為什麼選擇 Netlify？

  • 免費，支援自動部署
  • 無需設定伺服器，自動 HTTPS
  • 與 GitHub 深度整合
  • 自動回滾和版本管理
  • 全球 CDN，訪問速度快

部署步驟（非常簡單）

Step 1：在 Antigravity 中完成開發

確保代碼能正常運行，所有測試通過。

Step 2：推送到 GitHub

1. 在 Antigravity 右上角點擊 "Push to GitHub" 2. 授權 Antigravity 訪問你的 GitHub 3. 選擇倉庫或創建新倉庫 4. 點擊 Confirm

Step 3：在 Netlify 連接

  1. 訪問 https://netlify.com
  2. 用 GitHub 帳號登入
  3. 點擊 "New site from Git"
  4. 選擇你剛推送的倉庫
  5. 設定構建命令和輸出文件夾

Step 4：自動部署

完成！
Netlify 會自動：
✓ 監聽 GitHub 更新
✓ 每次 Push 時自動部署
✓ 生成唯一 URL（https://xxx.netlify.app）
✓ 配置 HTTPS 證書
✓ 設定自動備份

部署配置範例

（Netlify 自動檢測） React 項目： - Build command: npm run build - Publish directory: build Next.js 項目： - Build command: npm run build - Publish directory: .next Vue 項目： - Build command: npm run build - Publish directory: dist

部署後的管理

功能 說明
自定義域名 綁定自己的域名
環境變數 安全設定 API Key、數據庫連接
版本管理 回滾到之前的部署版本
Webhook 與第三方服務整合
分析 查看訪問量、性能指標

實戰建議

  • 第一次部署可能需要 5-10 分鐘，之後每次更新只需 1-2 分鐘
  • 如果部署失敗，檢查 Netlify Build 日誌找出問題
  • 環境變數要在 Netlify Dashboard 中設定，不要硬寫在代碼裡
  • 定期檢查依賴更新，保持安全性
