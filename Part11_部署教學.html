部署教學概述

什麼是部署？
部署是將你開發的網頁或應用程式發布到網路上，讓全世界的人都能透過網址訪問你的作品。

三種部署方案

Google Sites

最簡單的方式，將程式碼嵌入 Google 協作平台，適合教學展示

Netlify

專業的靜態網站託管，支援自訂網域，適合正式專案

Google Apps Script

連接 Google 試算表作為後端資料庫，實現資料串接

方案比較

特性 Google Sites Netlify GAS
難度 最簡單 中等 進階
適用場景 教學展示、快速分享 正式網站、作品集 需要後端資料
自訂網域 有限制 完全支援 不支援
費用 免費 免費方案可用 免費

Google Sites 部署教學

適用情境
將 Gemini Canvas 生成的網頁程式碼嵌入 Google 協作平台，輕鬆建立專業的教學或專案展示網站。

步驟一：準備並嵌入程式碼

1複製程式碼

Gemini 產生系統後，點擊 Canvas 右上角的「共用」→「複製內容」。這個步驟是複製 Canvas 的程式碼。

2前往 Google 協作平台

在瀏覽器搜尋「Google Sites」或直接前往 sites.google.com，開啟一個新網站或現有網站。

3新增協作平台網站

在上方選單點擊「+」號，新增一個協作平台（網站）。

4整頁嵌入

點擊右側的頁面，下方按「+」按鈕，選擇「整頁嵌入」，輸入頁面名稱，按下「完成」。

Google Sites 部署教學（續）

步驟二：設定嵌入內容

5選擇嵌入方式

在右側「插入」選單中，點擊「嵌入」→「嵌入程式碼」。

6貼上程式碼

將從 Gemini Canvas 複製的 HTML/CSS/JavaScript 程式碼貼入，按下「下一步」→「插入」。

步驟三：發布網站

7調整嵌入大小

拖曳嵌入區塊的邊緣，調整大小使其符合頁面。建議使用全寬度顯示。

8發布網站

點擊右上角「發布」按鈕，輸入網址名稱（例如：my-project），選擇公開或限定存取，按下「發布」。

小技巧
• 網址格式：https://sites.google.com/view/你的網址名稱
• 發布後可隨時修改內容，修改後記得重新發布
• 可設定「任何人都能檢視」讓所有人存取

Netlify 網站部署教學

Netlify 是什麼？
Netlify 是專業的靜態網站託管平台，提供免費方案、自訂網域、HTTPS 加密、自動部署等功能。適合正式專案和作品集。

步驟一：準備工作

1準備 HTML 檔案

在 Gemini Canvas 中，點擊右上角「程式碼」，按 Ctrl+A 全選程式碼。

2建立本地檔案

開啟記事本或 VS Code，貼上程式碼，另存為 index.html（檔案類型選「所有檔案」）。

3建立專案資料夾

在桌面建立一個資料夾（例如：my-website），將 index.html 放入資料夾。

步驟二：註冊 Netlify

4前往 Netlify

訪問 https://netlify.com，點擊「Sign up」註冊帳號（建議使用 GitHub 或 Google 帳號登入）。

Netlify 網站部署教學（續）

步驟三：部署網站

5進入儀表板

登入後進入 Netlify Dashboard，點選左側選單的「Projects」。

6手動部署

在「...or deploy manually」區域，將你的專案資料夾直接拖曳到上傳區域，或點擊「browse to upload」選擇資料夾。

7等待部署完成

Netlify 會自動處理部署，通常只需 10-30 秒。完成後會顯示一個隨機生成的網址。

步驟四：自訂網址

8修改網站名稱

進入 Site settings → Site details → Change site name，輸入你想要的名稱（例如：my-cool-project）。

完成！
你的網站已上線：https://你的名稱.netlify.app
• 免費 HTTPS 加密
• 全球 CDN 加速
• 支援自訂網域（付費方案）

Netlify 網站管理

更新網站內容

方法一：重新拖曳

  1. 修改本地的 HTML 檔案
  2. 回到 Netlify Dashboard
  3. 點擊「Deploys」分頁
  4. 將更新後的資料夾拖曳上傳

方法二：連結 GitHub

  1. 將專案推送到 GitHub
  2. 在 Netlify 選擇「Import from Git」
  3. 連結你的 GitHub 倉庫
  4. 之後每次 Push 自動部署

刪除網站

注意：刪除後無法復原
1. 進入 Site settings → General → Danger zone
2. 點擊「Delete this site」
3. 輸入網站名稱確認刪除

常見問題

問題 解決方案
網頁空白 確認檔案名稱是 index.html（小寫）
樣式跑掉 檢查 CSS 路徑是否正確，使用相對路徑
圖片不顯示 確認圖片已放入資料夾並使用正確路徑

Google Apps Script 設定教學

GAS 是什麼？
Google Apps Script (GAS) 是 Google 提供的雲端腳本平台，可以將 Gemini 產生的後端程式碼部署到 Google Cloud，實現與 Google 試算表的資料串接。

步驟一：建立試算表

1建立 Google 試算表

前往 sheets.google.com，建立一個新的試算表作為資料庫。

2設定欄位標題

在第一行輸入欄位名稱，例如：姓名、電話、Email、訊息、時間戳記。

步驟二：開啟 Apps Script

3進入 Apps Script

在試算表中，點擊「擴充功能」→「Apps Script」，開啟 Apps Script 編輯器。

4貼上程式碼

將 Gemini 產生的 GAS 程式碼貼入編輯器，取代原本的內容。

Google Apps Script 部署

步驟三：部署為網頁應用程式

5儲存專案

點擊「儲存」按鈕（或 Ctrl+S），為專案命名。

6部署

點擊「部署」→「新增部署作業」→ 選擇類型「網頁應用程式」。

7設定存取權限

• 執行身分：選擇「我」
• 誰可以存取：選擇「所有人」
按下「部署」

8授權存取

首次部署會要求授權，點擊「授權存取」→ 選擇你的 Google 帳號 → 允許權限。

步驟四：取得網址與串接

9複製部署網址

部署完成後會顯示「網頁應用程式網址」，複製這個網址。

10貼到前端程式碼

回到 Gemini 對話，將網址貼給 AI，完成後端資料庫的連線串接。
