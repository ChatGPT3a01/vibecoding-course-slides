Google Apps Script (GAS) 簡介

什麼是 Google Apps Script？

Google Apps Script 是基於 JavaScript 的雲端指令碼平台，直接整合 Google 生態：

  • Google Sheets、Docs、Gmail、Drive
  • 無需伺服器，自動擴展計算能力
  • 觸發器與排程自動執行任務
  • 支援外部 API 整合

為什麼選擇 GAS？

低成本

免費使用配額充足，適合中小企業

易上手

JavaScript 語法，無需額外環境

原生整合

與 Google 服務無縫連接

快速部署

即時執行，無需編譯

數據自動化流水線：製造亂源 (Step 1)

場景：月度銷售數據混亂

現實問題：各個業務員回傳的 Excel 格式不一致、有重複數據、缺失欄位，無法直接分析

自動化任務

1監控指定 Gmail 資料夾，偵測新銷售報表

2自動下載附件到 Google Drive

3轉換為標準 Google Sheet 格式

GAS 關鍵程式碼範例

function checkEmailAttachments() { const threads = GmailApp.search('label:sales-reports'); threads.forEach(thread => { const messages = thread.getMessages(); messages.forEach(message => { const attachments = message.getAttachments(); attachments.forEach(attachment => { // 存儲到指定資料夾 DriveApp.getFolderById('FOLDER_ID') .createFile(attachment); }); }); }); }
結果：所有銷售數據自動流入統一資料夾，格式一致

GAS 清洗腳本

function cleanSalesData() { const sheet = SpreadsheetApp.getActiveSheet(); const range = sheet.getDataRange(); const values = range.getValues(); // 去重：創建唯一訂單集合 const uniqueOrders = new Set(); const cleanedData = values.filter(row => { if (!uniqueOrders.has(row[0])) { uniqueOrders.add(row[0]); return true; } return false; }); // 清除並寫入清洗後的數據 sheet.clearContents(); sheet.getRange(1, 1, cleanedData.length, cleanedData[0].length) .setValues(cleanedData); }
自動化優勢：消除人工檢查的時間成本，確保數據一致性

數據自動化流水線：商業洞察 (Step 3)

AI 賦能數據分析

使用 Gemini API 自動生成商業洞察：

function generateInsights() { const sheet = SpreadsheetApp.getActiveSheet(); const data = sheet.getDataRange().getValues(); // 準備數據摘要 const summary = data.map(row => `訂單: ${row[0]}, 金額: ${row[1]}, 區域: ${row[2]}` ).join('\n'); // 呼叫 Gemini API const response = UrlFetchApp.fetch('https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent', { method: 'post', contentType: 'application/json', payload: JSON.stringify({ contents: [{ parts: [{ text: '分析以下銷售數據，識別趨勢和異常：\\n' + summary }] }], apiKey: 'YOUR_GEMINI_API_KEY' }) }); const result = JSON.parse(response.getContentText()); Logger.log(result.candidates[0].content.parts[0].text); }

自動生成報告內容

  • 銷售排名前三的區域
  • 異常高/低訂單識別

黃金 Prompt 公式應用

Prompt 結構化模板

【角色】 你是資深商業分析師 【背景】銷售團隊需要快速找出業務機會 【任務】分析以下銷售數據，提出可行建議 【數據】[CSV 格式的銷售數據] 【要求】 - 格式：JSON 結構 - 內容：3 個主要洞察 + 2 個行動建議 - 風格：專業但易理解 【範例輸出】 { "insights": ["區域 A 增長 45%", ...], "actions": ["加大區域 A 投入", ...] }

在 GAS 中應用

function buildSmartPrompt(data, context) { const prompt = `【角色】你是資深商業分析師 【背景】${context.background} 【任務】${context.task} 【數據】${JSON.stringify(data)} 【要求】${context.requirements}`; return prompt; // 傳入 Gemini API }

NotebookLM 簡介

什麼是 NotebookLM？

Google 推出的 AI 助手，專門設計用於深度分析長篇文件，將知識轉化為洞察與媒體：

文件理解

支援 PDF、文檔、網頁等多種格式的深度分析

智能提問

AI 生成相關問題，幫助提取關鍵洞察

音頻化

將複雜文件轉換為可收聽的 Podcast

協作研究

支援多人協作、版本管理、引用追蹤

適用場景

  • 法律文件快速理解
  • 研究論文內容提取
  • 技術文檔深度分析
  • 市場報告洞察生成
  • 學習內容摘要製作

從長篇文件到 5 分鐘 Podcast

NotebookLM 音頻化流程

上傳文件
→
AI 分析
→
提取重點
→
生成音頻

操作步驟

1進入 notebooklm.google.com

2建立新筆記本，上傳或貼入文件

3點擊「Generate」→「Podcast」

4AI 自動生成對話式音頻（2-5 分鐘）

5下載或分享音頻連結

使用案例：快速學習

將 50 頁的技術文檔轉成 5 分鐘 Podcast，可在通勤時間學習，節省 90% 的閱讀時間

實作練習：勞動基準法快速理解

練習目標

使用 NotebookLM 快速理解法律文件，在 5 分鐘內獲得核心知識

步驟說明

Step 1: 搜尋「勞動基準法全文 PDF」，下載官方文件
Step 2: 上傳到 NotebookLM，AI 自動分析法律條文
Step 3: 點擊「Generate」→「Podcast」，生成對話式音頻
Step 4: 提問「員工加班費如何計算？」，AI 直接給出答案

預期成果

  • 5 分鐘 Podcast：核心概念速覽
  • 提問功能：快速定位法律條文
  • 對比工具：理解不同條款的關聯
  • 可分享連結：團隊快速培訓
結論：NotebookLM 讓複雜文件變簡單，是高效學習和知識提取的利器

AI Agent 實戰：SWE-Agent（開源）

什麼是 SWE-Agent？

由 Princeton 大學開發的開源 AI 代理人，能夠自動修復軟體工程問題：

自動編碼

根據 Issue 描述自動定位和修復程式碼

工具集成

整合 Git、命令行、代碼搜尋等工具

推理能力

多步驟推理，模擬真實開發工作流程

開源免費

可本地部署，無第三方依賴

工作流程

Issue: 「Function getUserData() 返回 undefined」 SWE-Agent 流程： 1. 搜尋 getUserData 函數定義 2. 分析調用棧和錯誤日誌 3. 識別缺失的參數驗證 4. 自動生成修復程式碼 5. 運行測試驗證修復 6. 提交 Pull Request

什麼是 Google Jules？

深度集成

企業級安全

典型應用
